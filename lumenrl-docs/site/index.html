// capabilities

Key Features

Everything you need for high-performance RL post-training on AMD GPUs

⚡

AMD-Native

AITER kernels (ASM > CK > Triton), MORI device-side RDMA collectives, and a ROCm-first stack from fused ops to MoE expert dispatch.

⚙

FP8 End-to-End

FP8 W8A8 rollout plus FP8 training with blockwise, MXFP8, or per-tensor scaling. Rollout correction via TIS/MIS for train-inference alignment.

▲

MoE-Stable RL

R3 (Rollout Routing Replay) captures inference routing distributions and replays them during training to keep MoE routers aligned.

♺

Flexible Backends

FSDP2 or Megatron-Core for training with Lumen FP8. ATOM for rollout with tensor, expert, and data parallelism.

☷

GRPO • DAPO • PPO

Clipped surrogate, asymmetric clip, and value-based objectives with group-relative baselines and GAE-Lambda.

⚗

Production Ready

Ray orchestration, YAML configs, multi-node SLURM launch, Docker images, and checkpoint/resume for long training runs.

// get started

Quick Start

Docker or editable install, then launch with BF16, FP8, or MoE R3 configs

install
# Clone with submodules
git clone --recursive \
  https://github.com/ZhangDanyang-AMD/Lumen-RL.git
cd Lumen-RL

# Docker build
bash docker/build.sh

# Or editable install
pip install -e ".[dev]"
pip install -e third_party/Lumen
pip install -e third_party/verl
run
# BF16 baseline (single node, 8 GPU)
python examples/run_grpo.py \
  --config configs/grpo_dense_bf16.yaml

# FP8 end-to-end
python examples/run_grpo.py \
  --config configs/grpo_dense_fp8.yaml

# MoE + R3 + FP8
python examples/run_grpo.py \
  --config configs/grpo_moe_fp8_r3.yaml
fp8 config
quantization:
  rollout:
    precision: fp8
    use_deep_gemm: true
  training:
    fp8: hybrid
    fp8_recipe: blockwise
    fp8_weight_cache: true
  rollout_correction:
    enabled: true
    method: tis
    clip: 1.5
moe r3 config
moe:
  r3:
    enabled: true
    record_router_logits: true
    replay_mode: distribution

# DAPO example
bash examples/DAPO/run_1a_bf16_baseline.sh
bash examples/DAPO/run_1d_fp8_e2e_blockwise.sh

// experiments

FP8 Alignment Experiments

DAPO on Qwen3-8B-Base, 8× MI350X — reproducing VERL FP8 benchmarks

Experiment 1A — BF16 Baseline

Pure BF16 training + BF16 rollout. Reference curve for all FP8 comparisons.

bash examples/DAPO/run_1a_bf16_baseline.sh

Experiment 1B — FP8 Rollout + TIS

BF16 training + FP8 rollout with token-level importance sampling. Expected: 1B ≈ 1A.

bash examples/DAPO/run_1b_fp8_rollout_tis.sh

Experiment 1D — FP8 E2E Blockwise

Full Lumen FP8 training (blockwise 128) + FP8 rollout + TIS. Priority 1 — closest to TE reference.

bash examples/DAPO/run_1d_fp8_e2e_blockwise.sh

Experiment 1F — FP8 E2E Per-Tensor

Lumen FP8 per-tensor delayed scaling. Coarsest granularity — validates lower bound.

bash examples/DAPO/run_1f_fp8_e2e_per_tensor.sh
